# VibeTrack UWB Prototype

VibeTrack is now focused on Apple Nearby Interaction between an iPhone and a
Qorvo DWM3000EVB-based accessory.

The previous repo version used two ESP32 + DWM3000 boards for
DWM3000-to-DWM3000 ranging and sent JSON telemetry to the iPhone over BLE. That
prototype has been replaced. This version targets iPhone-to-accessory Nearby
Interaction, where real distance and direction come from Apple's
`NearbyInteraction` framework after the accessory provides valid Apple/Qorvo
configuration data.

## Current Architecture

1. iPhone app
   - Scans for the BLE accessory named `VibeTrack-UWB`.
   - Connects with CoreBluetooth.
   - Receives accessory Nearby Interaction configuration data over BLE.
   - Creates `NINearbyAccessoryConfiguration`.
   - Runs `NISession`.
   - Displays only real `NINearbyObject.distance` and `NINearbyObject.direction`.

2. ESP32 + DWM3000EVB firmware
   - Starts BLE advertising as `VibeTrack-UWB`.
   - Exposes the BLE service and characteristics used by the iPhone app.
   - Initializes ESP32 SPI with `SPI.begin(18, 19, 23, 4)`.
   - Resets the DWM3000EVB on `GPIO27`.
   - Reads and prints the DWM3000/DW3110 device ID.
   - Reports that Qorvo Apple NI accessory firmware is missing.

3. Missing production piece
   - Apple Nearby Interaction with third-party accessories requires real
     Apple/Qorvo-compatible accessory firmware.
   - Generic Arduino DW3000 ranging code is not enough.
   - The ESP32 sketch does not invent accessory configuration, distance, or bearing.

## BLE Protocol

Device name: `VibeTrack-UWB`

| Purpose | UUID |
| --- | --- |
| NI BLE service | `7E400001-B5A3-F393-E0A9-E50E24DCCA9E` |
| Accessory NI configuration notify/read/write | `7E400011-B5A3-F393-E0A9-E50E24DCCA9E` |
| iPhone NI configuration write | `7E400012-B5A3-F393-E0A9-E50E24DCCA9E` |
| Status/debug notify/read/write | `7E400013-B5A3-F393-E0A9-E50E24DCCA9E` |

The accessory configuration characteristic must eventually carry real accessory
configuration bytes produced by Qorvo's Apple NI-compatible accessory stack.
The current ESP32 Arduino sketch leaves this empty and reports:

```text
NI accessory firmware not implemented because Qorvo Apple NI accessory API is missing.
```

## Real Now

- BLE scanning and connection from iPhone to ESP32.
- BLE GATT characteristics for the Apple NI data channel.
- iOS `NISession` code path that starts only after real accessory config bytes are received.
- Real `NINearbyObject.distance` and `direction` display code.
- DWM3000EVB SPI reset and DEV_ID read on ESP32.
- Honest missing-firmware status from the accessory sketch.

## Requires Qorvo/Apple NI Firmware Later

- Valid accessory configuration bytes for `NINearbyAccessoryConfiguration(data:)`.
- Accessory-side Apple Nearby Interaction UWB protocol behavior.
- Consuming the iPhone shareable NI configuration bytes generated by `NISession`.
- Real iPhone-to-DWM3000EVB ranging and direction updates.

One DWM3000EVB can be enough for iPhone Nearby Interaction only when the
accessory firmware implements the Apple/Qorvo-compatible NI accessory protocol.
DWM3000-to-DWM3000 Arduino ranging examples do not provide that protocol.

## Quick Test

1. Flash `firmware/VibeTrack_NI_Accessory_ESP32/VibeTrack_NI_Accessory_ESP32.ino`.
2. Open Serial Monitor at `115200`.
3. Confirm SPI starts and a DWM3000 DEV_ID is printed.
4. Confirm BLE advertises as `VibeTrack-UWB`.
5. Build the iOS source on a physical iPhone 16 Pro Max.
6. Allow Bluetooth and Nearby Interaction permissions.
7. Scan and connect.
8. Press Start Nearby Interaction.
9. With the current ESP32 Arduino firmware, expect `Accessory configuration missing`.
10. After installing real Qorvo NI accessory firmware, expect real `NINearbyObject` updates.

See `docs/` for setup, wiring, and troubleshooting details.
